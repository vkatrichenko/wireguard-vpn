# Tasks: Runtime Webhook Management

- **Functional Spec:** [functional-spec.md](./functional-spec.md)
- **Technical Spec:** [technical-considerations.md](./technical-considerations.md)
- **Stack:** Go std-lib dashboard (`dashboard/`) — refactor `internal/notify`, new server write handlers + About-tab card, `main.go` wiring; **plus** a Terraform boot-seed layer (`terraform/modules/wireguard/*` + `terraform/dev/`). Go sub-tasks → `go-fullstack`; Terraform/user-data sub-tasks → `terraform-aws` / `linux-cloud-init`.

**Confirmed decisions** (from the 008 design + tech spec):

- **Env/SSM authoritative at boot; UI override is runtime-only and never persisted.** `DASHBOARD_WEBHOOK_URL` seeds the holder at startup; the UI can override it in memory; a restart re-seeds and discards the override. The secret is never written to SQLite or disk.
- **No auth** (VPN trust): any peer can view the masked URL and set/test/revert. First **write** endpoints on the dashboard — a deliberate, bounded exception to the read-only posture (outbound-target only; no inbound surface).
- **https-only** webhook validation; **masked** display (scheme+host via `redactURL`); URL never logged in full, never rendered in full.
- **Controls:** Set + Test + Revert, on the **About tab**.
- **Two shippable layers:** (A) Go runtime-management (Slices 1–4) works with any env seed; (B) Terraform SSM→`EnvironmentFile` wiring (Slice 5) supplies a real deployment's boot seed — and also lands the **007 alert env knobs** that were deferred from spec 007.

**Grounding (existing seams):** `internal/notify` has `Notifier` (`Notify(ctx, msg) error`), `*Webhook` (fixed URL + timeout/retry/redaction), `NoOp`, and `redactURL`. The poller's `runDispatchLoop` already calls `Notifier.Notify` off the critical path. 007 Slice 5 added `alerts.StatusHolder` with a one-shot `SetEnabled` (becomes dynamic here). `main.go` builds the notifier and threads it into `poller.New`/`server.New`. Terraform's server-key secret pattern (`modules/wireguard/datasource.tf` `data "aws_ssm_parameter"` → `locals.tf` `templatefile`) is the template for the webhook seed; the systemd unit lives in `templates/user-data.txt`; the instance has `lifecycle { ignore_changes = [user_data] }`.

Each slice leaves the dashboard runnable; in-session proof is `go test` (+ `-race`) + httptest. The Terraform slice is proven by `fmt`/`validate`/`tflint`/`trivy` + a reviewed `plan` (owner applies). Real webhook receipt is the manual Slice 6.
