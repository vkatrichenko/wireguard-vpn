# Tasks: Geo Map Zoom & Legibility

- **Functional Spec:** [functional-spec.md](./functional-spec.md)
- **Technical Spec:** [technical-considerations.md](./technical-considerations.md)
- **Stack:** Front-end only — `dashboard/web/static/world-map.js`, the geo-map card in `web/templates/tabs/clients.html`, geo CSS in `web/static/app.css`, JS unit tests in `dashboard/jstest/world-map.test.mjs`. **No backend / `/api/geo` / geo-data change.** All sub-tasks → `go-fullstack` (applying the `frontend-design` skill for the controls).

**Confirmed decisions** (from the 010 design):

- **Defect fixes ship first** (independent, immediate): the oversized marker (`background` shorthand clobbering `background-clip` → 44px blob) and the empty-state that can show alongside a marker.
- **Zoom = CSS `transform` on a `.geo-canvas` wrapper** (basemap `<img>` + `.geo-markers` overlay inside an `overflow:hidden` viewport) — markers ride the transform (percent-positioned in the canvas), **dots counter-scaled** so they stay constant size. No tiles, fully offline.
- **Controls-first, then gestures:** `+`/`−`/reset buttons (works everywhere, keyboard-accessible) before wheel/pinch/drag.
- **Bounded** `z ∈ [1, ~8]` (1 = fit), pan clamped; `prefers-reduced-motion`-gated; controls styled with the spec-009 tokens. The transform lives on `.geo-canvas` so the 10s `/api/geo` tick doesn't reset the view.

**Grounding (existing seams):** `world-map.js` has pure `project`/`projectPercent`/`clusterPeers`, `renderMarkers` (absolute %-positioned `<button>` markers over `<img class="geo-basemap" src="/static/world.svg">`), `applyMarkerColor` (the `style.background` bug), the empty toggle (`emptyEl.hidden = clusters.length > 0`, success-path only), and a 10s `/api/geo` tick. The geo card: `.geo-map-frame` (`aspect-ratio:2/1` + `max-width`) > basemap + `.geo-markers` + `.geo-empty`, with the legend/caption as siblings. `jstest/world-map.test.mjs` runs `node --test` from inside `jstest/` (a Node directory-runner quirk — not from the parent).

Each slice leaves the dashboard runnable; automated proof = `jstest` pure-helper tests + Go render tests + offline grep + build. The zoom/pan/gesture *visual* behavior is an operator manual gate (no browser MCP). Do NOT touch anything outside the geo map.
